#+title: notmuch-bone.el --- Highlight BARK reports in notmuch

[[https://intver.org/][https://img.shields.io/badge/versioning-intver.org-blue.svg?style=for-the-badge]]

=notmuch-bone= highlights notmuch emails whose =Message-Id= matches an
open [[https://codeberg.org/bzg/bark][BARK]] report. It reads the same =reports.json= sources as [[https://codeberg.org/bzg/bone][bone]].

[[file:notmuch-bone-screenshot.png]]

* Requirements

- The [[https://codeberg.org/bzg/bone.el][bone.el]] library, which provides the shared data layer
  (configuration, report cache and =state.edn= handling).  Once
  =notmuch-bone= is on MELPA this dependency will be pulled in
  automatically; until then, see /Installation/ below.
- =~/.config/bone/config.edn= (shared with bone), containing at least
  a =:sources= vector with =:url= entries pointing to =reports.json=
  files (local paths or URLs).
- =notmuch-bone= supports =reports.json= bark-format =0.9.1=.
- =notmuch-bone= does not call the =bone= binary, it only reads the
  files that =bone= produces.  You need to have run =bone= at least
  once to generate =config.edn= and the =reports.json= files.

* Installation

=notmuch-bone= is not on MELPA yet.  On Emacs 30 or later, install it
from git with the built-in =package-vc=.  Install [[https://codeberg.org/bzg/bone.el][bone.el]] *first*,
since it is its dependency and is not in any package archive (the
=notmuch= dependency is pulled from NonGNU ELPA automatically):

#+begin_src emacs-lisp
(use-package bone
  :vc (:url "https://codeberg.org/bzg/bone.el" :rev :newest))

(use-package notmuch-bone
  :vc (:url "https://codeberg.org/bzg/notmuch-bone" :rev :newest))
#+end_src

Or interactively, in order:

: M-x package-vc-install RET https://codeberg.org/bzg/bone.el RET
: M-x package-vc-install RET https://codeberg.org/bzg/notmuch-bone RET

Update later with =M-x package-vc-upgrade=.

* Commands

- =M-x notmuch-bone= — Search for open BARK reports and highlight them.
- =M-x notmuch-bone-tree= — Show open BARK reports in =notmuch-tree=.
- =M-x notmuch-bone-highlight= — Highlight matches in current
  =notmuch-search= buffer.
- =M-x notmuch-bone-topic= — Filter reports by selected topic.
- =M-x notmuch-bone-clear= — Remove all =notmuch-bone= overlays.

** Local marks

The following commands toggle bone's local marks. They edit
=~/.config/bone/state.edn=, which is shared with the =bone= CLI and
=gnus-bone=, so marks are visible across all three.

- =M-x notmuch-bone-mark-read= — Toggle the =:read-at= timestamp.
- =M-x notmuch-bone-mark-todo= — Toggle the =:todo= flag.
- =M-x notmuch-bone-mark-sticky= — Toggle the =:sticky= flag.

The annotation gains a leading column showing the current mark:
=!= for =:todo=, =*= for =:sticky=, =r= for =:read-at= (without flag).

In =notmuch-search-mode= a line is a thread; the *first* message-id in
that thread that is also a BARK report is the one toggled.  Use
=M-x notmuch-bone-tree= to mark a specific report when a thread contains
several.
